This round's fixes — checklist

✓ Bar — actual root cause found, not just re-checked
  • The v6 CSS hid scrollbars on .scroll-area and .scroll-x, but not on .tabs — and .tabs also has overflow-x:auto. On a desktop webkit preview that's invisible either way; on your phone's real browser/webview the tab row's native horizontal scrollbar renders as a solid gray track — that's the bar you kept seeing under Upcoming/Live/Results/Other Tours
  • Fixed by adding .tabs to the same hide-scrollbar rule (scrollbar-width:none + ::-webkit-scrollbar{display:none}) — same fix will need to land in the real component's ScrollView props (showsHorizontalScrollIndicator={false}) when this gets implemented, it's a one-line prop already used elsewhere in the app
✓ Live cards — single line, ~4× smaller
  • Old hero card was ~90px tall; new .live-row is ~26px — one score line, dot, optional event tag for cross-tour matches
  • 12 live matches shown fitting in view with minimal scroll; only the single "your tournament's" match gets the slightly larger "featured" treatment, everything else (Also Live / other tours) is a plain compact row
✓ Draw — real connected bracket, not a card list
  • Researched standard tournament-bracket UI patterns (Challonge, ESPN, react-tournament-brackets) — the defining trait missing before was connector lines linking each pair to its next-round slot, with the next match vertically centered between its two feeders
  • Cards are now absolutely positioned on a coordinate grid with an SVG line overlay drawing the actual bracket tree — colors/card styling unchanged (still real #252525/#FFA726), only the layout mechanism changed from a flowing list to a positioned tree
  • Real implementation would compute these Y-positions from roundFormats/match count per round rather than hardcoded pixels used here for the mockup
